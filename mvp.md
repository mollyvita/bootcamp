# План работы

Веб-приложение - агрегатор мероприятий с различных вузов для студентов/абитуриентов
- Само приложение - стандартный HTML+CSS+JS

# Пользовательский путь записи на мероприятие:
- Пользователь открывает сайт
- Пользователь видит множество мероприятий и студенческих клубов от разных университетов разных видов
- Пользователь может отфильтровать мероприятия по конкретному вузу, виду мероприятий (дни открытых дверей, клубы по интересам) и тому, кем он является сейчас (студент или абитуриент)
- Пользователь выбирает конкретное мероприятие
- Открывается простая форма регистрации с ФИО, почтой и указанием, кем является регистрирующийся
- После регистрации пользователь видит в своем личном кабинете на какие мероприятия он записан
- На сайте также есть ИИ-ассистент, который рекомендует мероприятия на основе 

# Ограничения
- Это MVP, нужно грамотно показать один пользовательский путь по регистрации на конкретное мероприятие/клуб с отображением этой регистрации впоследствии
- Так как это MVP, желателен, но не критичен крутой дизайн
- В example_ai.py находится пример работы с API для ИИ-ассистента, используй только его
- В конце поднимай сервер сразу, даже не спрашивай меня об этом
- Авторизация будет добавлена позже, сделай вид, что там уже есть некий юзер, из-под которого мы зашли
